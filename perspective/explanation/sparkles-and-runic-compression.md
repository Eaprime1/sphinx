# Sparkles and Runic Compression

When a document's journey is complete — when it has moved through the {term}`Aperture`,
deepened inside a {term}`Gravity Well`, and reached terminal state — it does not simply
disappear. It becomes a {term}`Sparkle`.

## What a Sparkle is

A Sparkle is the compressed essence of a completed knowledge journey. It is small, dense,
and permanently encoded. It does not function as a working document — you do not edit a
Sparkle. You reference it. It is the crystallized record of what was, how it moved, and
what it became.

Think of a Sparkle as a rune: a symbol that carries compressed meaning, readable by those
who know the language.

## Runic compression

The {term}`Runyatr` is the encoding system that turns a completed document into a Sparkle.
It is "runic" because:

1. **It is compact.** A Sparkle encodes the full journey of a document in a small structure,
   not a large one.
2. **It preserves provenance.** The Runyatr records who authored the document, which AI
   partners contributed, when it entered each well, and when it was sparklized.
3. **It is readable by AI.** The encoding is designed so that AI models can parse a Sparkle
   and reconstruct the journey history without needing the original documents.

## The Monarch empty vector

The {term}`Monarch` is the null Sparkle — the empty vector. It represents the state of
a journey that was initiated but produced no sparklizable content. Not every journey
produces a Sparkle. The Monarch records the attempt without fabricating an artifact.

The Monarch is important because it prevents the system from losing track of dead ends.
Dead ends are information too.

## Sparklization process

A document becomes a Sparkle when:

1. It has reached the {term}`Void` well (terminal compression state).
2. A human or automated trigger initiates sparklization.
3. The Runyatr encodes the journey metadata into the Sparkle structure.
4. The original document is archived (not deleted — it remains accessible but is no longer
   the primary artifact).
5. The Sparkle is indexed in the {term}`Gravity Well`'s Sparkle registry.

## In the documentation system

This doc set treats its own completed reference pages as candidates for sparklization
when they reach a stable, permanent form. A sparklized page would carry:

```yaml
sparkle_id: SPK-XXXX
sparklized: "2026-01-01"
runyatr_code: "GW:REF:APERTURE:v1.2"
original_path: reference/entities/aperture.md
```
