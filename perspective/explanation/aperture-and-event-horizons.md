# Aperture and Event Horizons

The {term}`Aperture` is the most important concept in Perspective. It is the threshold —
the deliberate act that separates uncertain from canonical, pool from well, draft from
permanent record.

## What the Aperture is

In optics, an aperture controls how much light passes through a lens. In Perspective, the
Aperture controls how much knowledge passes from a {term}`Pool` into a {term}`Gravity Well`.

It is not automatic. Crossing the Aperture is a decision. Something must meet a threshold
of readiness — enough pressure, enough clarity, enough intentionality — before it passes
through.

## Pressure and thresholds

Content in a Pool accumulates *pressure* as it develops:

- References from other documents increase pressure.
- Repeated engagement by AI or human authors increases pressure.
- Explicit tagging as "ready" increases pressure.

When pressure exceeds the well's threshold, the Aperture can open. The default threshold
is a human decision, but automation (Zapier, IFTTT) can handle the mechanics once the
threshold is defined.

## The ethics of flow

Not everything should pass through the Aperture. Some content is:

- **Liminal by nature**: brainstorming, half-formed questions, exploratory notes. These
  belong in the Pool. Forcing them through the Aperture produces false permanence.
- **Time-bound**: notes from a meeting, a draft email. These may belong in a
  {term}`Pulsar` well, not a {term}`Gravitar`.
- **Private**: some content should not graduate. The Aperture also guards against
  premature exposure.

The ethics of the Aperture: *only graduate what is ready to be permanent.*

## Event horizons

Each well type has a different event horizon — the point of no return:

| Well type | Event horizon character |
|---|---|
| Gravitar | Hard. Content that enters rarely leaves. |
| Magnetar | Permeable. Content flows in and out during active phase. |
| Pulsar | Cyclic. The horizon opens and closes on rhythm. |
| Void | Absolute. Nothing returns from the Void. |

## In the documentation system

This doc set uses the Aperture metaphor for its own workflow:

- **Pool** → `play/` directory (drafts, stubs, experiments)
- **Aperture** → the editorial review: does this page belong in a stable quadrant?
- **Gravity Well** → `tutorials/`, `how-to/`, `explanation/`, `reference/`

See {doc}`/ai-playbook` for the checklist used when graduating content from `play/`.
