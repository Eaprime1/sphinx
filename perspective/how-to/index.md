# How-to Guides

Sharp, task-oriented recipes. Each guide has a clear goal and success criteria.
Unlike tutorials, how-to guides assume you already understand the architecture —
they focus on *doing*, not *learning*.

For the conceptual background behind any recipe, see {doc}`/explanation/index`.

## Planned guides

**How to add a new gravity well and wire it into the Aperture routing**
: Given an existing pool on Google Drive, this guide walks you through creating a new
  well in Box, instantiating an Aperture between them, and verifying that pressure
  scoring is active.

**How to use AI to refactor a messy folder into pools and receptacles**
: Step-by-step recipe for taking an unstructured Google Drive folder and classifying
  its contents into pool items with appropriate pressure scores and target wells.

**How to auto-generate API docs from docstrings with sphinx-autodoc**
: For projects with Python codebases that live alongside Perspective docs — wire
  autodoc into the build and generate reference pages from docstrings.

**How to graduate a page from play/ to a stable quadrant**
: The editorial checklist (from {doc}`/ai-playbook`) expressed as a step-by-step recipe.
