# Glossary

All domain terms used in Perspective documentation. Reference any term from other pages
with the `:term:` role, e.g. `` {term}`Gravity Well` ``.

```{glossary}
Gravity Well
   A stable knowledge attractor. Content flows toward a gravity well, organizes around it,
   and deepens over time. Wells live in Box. See {doc}`explanation/gravity-well-cosmology`
   and {doc}`reference/entities/gravity-well`.

Pool
   A liminal holding space for content before it crosses the Aperture. Pools live in
   Google Drive. Content in a pool is a candidate, not a commitment.
   See {doc}`reference/entities/pool`.

Aperture
   The threshold between a Pool and a Gravity Well. Crossing the Aperture is a deliberate
   act — it marks the transition from uncertain to canonical. The Aperture has pressure
   thresholds and ethics flags. See {doc}`explanation/aperture-and-event-horizons` and
   {doc}`reference/entities/aperture`.

Sparkle
   The compressed essence of a completed knowledge journey. A Sparkle is permanent,
   AI-readable, and encoded with Runyatr. It is the crystallized record of what a document
   was and how it moved. See {doc}`reference/entities/sparkle`.

Runyatr
   The runic encoding system that compresses completed documents into Sparkles. Runyatr
   codes are structured strings encoding well type, quadrant, concept, and version.
   Readable by AI models to reconstruct journey history.

Monarch
   The null Sparkle — the empty vector. A Monarch records a knowledge journey that was
   initiated but produced no sparklizable content. Prevents re-attempting empty journeys.

Gravitar
   A Gravity Well type. The strongest attractor — dense, foundational, long-lived. Content
   that enters rarely leaves. See {doc}`explanation/gravity-well-cosmology`.

Magnetar
   A Gravity Well type. Intensely active — high inbound pull and high outbound emission.
   Burns bright during project phases; can collapse into a Gravitar at completion.
   See {doc}`explanation/gravity-well-cosmology`.

Pulsar
   A Gravity Well type. Rhythmic — activates on a cycle and goes quiet between pulses.
   Meeting notes, sprint reviews, and recurring reports live in Pulsar wells.
   See {doc}`explanation/gravity-well-cosmology`.

Void
   A Gravity Well type. Absorbing only — content enters and is held in deep compression.
   Nothing emerges from the Void. It is the terminal state before Sparklization.
   See {doc}`explanation/gravity-well-cosmology`.

Sparklization
   The process of compressing a completed document into a Sparkle via the Runyatr system.
   Initiated when content reaches the Void well state. See
   {doc}`explanation/sparkles-and-runic-compression`.

Ka Signature
   The unique runic identifier for a Gravity Well, encoding its type, creation context,
   and lineage. Each Ka signature is unique within the Perspective system.

Diátaxis
   The documentation framework used to structure this doc set. Four quadrants: Tutorials
   (learning), How-to guides (tasks), Explanations (understanding), Reference (definitions).
   See `https://diataxis.fr/`.

MyST
   Markedly Structured Text — a Markdown dialect that supports Sphinx directives and roles.
   The authoring format for this documentation set. AI-friendly and human-readable.
```
